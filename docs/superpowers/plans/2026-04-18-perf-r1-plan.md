# Perf R1 — Partition Scale-Out + Batch Cap Sweep · Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Execute the perf-R1 experiment from `docs/superpowers/specs/2026-04-18-perf-r1-partition-scale-out-design.md` (F095). Push `perf_check.sh --shm` median write throughput from the F094 baseline of **54.5 k ops/s** to **≥ 100 k** (Tier A) or at least **80 k** (Tier B), with *no* code changes to PS / Stream / RPC hot paths. Decide on merge to `compio` only after user review of results.

**Architecture:** This is an **experiment**, not a feature. All writes are:
1. One pre-flight check (Task 0) that compio's runtime backend is genuinely io_uring, not a silent epoll / polling fallback — without this, the ceiling comparison is meaningless.
2. One surgical runtime knob (`AUTUMN_GROUP_COMMIT_CAP` env var, replaces the compiled constant `MAX_WRITE_BATCH`).
3. One perf-check correctness fix (multi-partition read routing) required for the matrix to be valid.
4. Three orchestration extensions (`perf_check.sh --partitions`, `cluster.sh --3disk`, `cluster.sh --multidisk-1node`).
5. One throwaway sweep driver (`scripts/perf_r1_sweep.sh`).
6. 5 phases of measurement (A1 → A2 → A3 (conditional) → A4 → A5) with **review checkpoints between each phase** so the user can stop early on Tier A or pivot on Tier C.

**Tech Stack:** bash (scripts), Rust 2021 with `compio` runtime (existing PS + CLI), `autumn-rpc` binary protocol, `rkyv` for control-plane encoding. No new dependencies.

**Worktree:** All work in `/data/dongmao_dev/autumn-perf-r1` on branch `perf-r1-partition-scale-out`. The main worktree at `/data/dongmao_dev/autumn` stays on branch `compio` and MUST NOT be touched. Every command below assumes `cd /data/dongmao_dev/autumn-perf-r1/autumn-rs` unless stated otherwise (Rust crate root).

**Spec simplification discovered during plan recon:**
The spec (§4.2 item 5) listed `presplit.sh` as a separate throwaway helper. Plan recon found that an equivalent mechanism already exists: `autumn-client bootstrap --presplit N:hex1,hex2,...` (via `hex_split_ranges()`), invoked by `cluster.sh` when `AUTUMN_BOOTSTRAP_PRESPLIT` is set. So this plan skips `presplit.sh` entirely and wires `perf_check.sh --partitions` directly to the existing env. The spec §4.2 item 5 is effectively **obsoleted** by the existing bootstrap path.

**Baseline facts (gathered at plan-writing time, DO NOT re-derive):**
- Compiled group-commit cap: `const MAX_WRITE_BATCH: usize = WRITE_CHANNEL_CAP * 3;` where `WRITE_CHANNEL_CAP = 1024`, giving **3072** requests. Secondary cap: `MAX_WRITE_BATCH_BYTES = 30 MB`. Defined in `crates/partition-server/src/lib.rs:45-47`. Used in `lib.rs:348`, `lib.rs:366`, `background.rs:364`.
- Pre-split wire protocol already exists: `autumn-client bootstrap --presplit N:hexstring` (via `hex_split_ranges()` at `crates/server/src/bin/autumn_client.rs:22`). `cluster.sh` exposes it via env `AUTUMN_BOOTSTRAP_PRESPLIT` at `cluster.sh:164`. Format: `"N:mid1,mid2,...,midN-1"` with each midpoint an 8-char lowercase hex string (e.g. `"4:3fffffff,7ffffffe,bffffffd"` for 4 partitions).
- ExtentNode multi-disk already supported (F021): `autumn-extent-node --data /d1,/d2,/d3` with `ExtentNodeConfig::new_multi()` at `crates/server/src/bin/extent_node.rs:101`.
- **Known bug** revealed during plan recon: `perf-check`'s multi-partition read path breaks because keys `pc_{tid}_{seq}` all lexically belong to the LAST partition (they start with `'p'` = `0x70`, above any hex midpoint), but writes round-robin by `tid % N`, landing data in the "wrong" partition. Read-phase `resolve_key` routes to the last partition and finds nothing. Task 4 fixes this by tracking `(key, part_id, ps_addr)` from write phase and reusing the mapping in the read phase — no reliance on `resolve_key` for the pc_ keyspace.

```rust
const WRITE_CHANNEL_CAP: usize = 1024;
const DEFAULT_MAX_WRITE_BATCH: usize = WRITE_CHANNEL_CAP * 3;

/// Group-commit request count cap. Read once from env `AUTUMN_GROUP_COMMIT_CAP`
/// if set to a positive integer in [1, 1_000_000]; otherwise falls back to
/// DEFAULT_MAX_WRITE_BATCH (3072). See docs/superpowers/specs/2026-04-18-*.md.
fn max_write_batch() -> usize {
    static CELL: std::sync::OnceLock<usize> = std::sync::OnceLock::new();
    *CELL.get_or_init(|| {
        std::env::var("AUTUMN_GROUP_COMMIT_CAP")
            .ok()
            .and_then(|s| s.parse::<usize>().ok())
            .filter(|&n| n > 0 && n <= 1_000_000)
            .unwrap_or(DEFAULT_MAX_WRITE_BATCH)
    })
}
```

## Task 8 · `perf_r1_sweep.sh` matrix driver (throwaway)

**Files:**
- Create: `autumn-rs/scripts/perf_r1_sweep.sh`

- [ ] **Step 1: Author the driver**

Create the file with this exact content:

```bash
#!/usr/bin/env bash
# perf_r1_sweep.sh — F095 Round 1 matrix driver (throwaway).
#
# Env:
#   PHASE         A1 | A2 | A3 | A4 | A5                           (required)
#   PARTITIONS    space-separated list of partition counts         (default: "1 2 4 8")
#   CAPS          space-separated group-commit caps, or empty for default
#                                                                   (default: "")
#   STORAGE_MODE  shm | 3disk | multidisk-1node                    (default: shm)
#   REPS          integer >= 1                                     (default: 3)
#
# Per-combination flow:
#   1. ./cluster.sh clean (and reset to chosen MODE + REPLICAS)
#   2. AUTUMN_GROUP_COMMIT_CAP=<cap> ./perf_check.sh <mode-flag> --partitions <N>
#      (without --update-baseline; baseline writes happen only in the final
#       wrap-up commit, not here)
#   3. Parse write_ops / read_ops / p99s from the perf-check stdout
#   4. ps -o pcpu for autumn-ps and (first) autumn-extent-node mid-run
#   5. Append one row to scripts/perf_r1_results.csv

set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
REPO_DIR="$(cd "$SCRIPT_DIR/.." && pwd)"
CSV="$SCRIPT_DIR/perf_r1_results.csv"

PHASE="${PHASE:?PHASE env var required}"
PARTITIONS="${PARTITIONS:-1 2 4 8}"
CAPS="${CAPS:-}"
STORAGE_MODE="${STORAGE_MODE:-shm}"
REPS="${REPS:-3}"

case "$STORAGE_MODE" in
    shm)                CLUSTER_FLAGS="";                 REPLICAS=3 ;;
    3disk)              CLUSTER_FLAGS="--3disk";          REPLICAS=3 ;;
    multidisk-1node)    CLUSTER_FLAGS="--multidisk-1node"; REPLICAS=1 ;;
    *) echo "unknown STORAGE_MODE=$STORAGE_MODE" >&2; exit 1 ;;
esac

PERF_FLAGS=""
[[ "$STORAGE_MODE" == "shm" ]] && PERF_FLAGS="--shm"

# Ensure CSV has a header.
if [[ ! -f "$CSV" ]]; then
    echo "phase,storage_mode,partitions,group_commit_cap,rep,write_ops,read_ops,write_p50_ms,write_p99_ms,read_p50_ms,read_p99_ms,ps_cpu_pct,ext_cpu_pct,notes" > "$CSV"
fi

parse_field() {
    local log="$1" label="$2" field="$3"
    awk -v L="$label" -v F="$field" '
        $0 ~ L" .*"F {
            for (i=1; i<=NF; i++) if ($i == F":") { print $(i+1); exit }
        }' "$log"
}

# Parse the output of `perf-check`'s print_bench_summary.
# Format is multi-line, label is "Write" or "Read".
extract_result() {
    local log="$1" label="$2"
    local ops p50 p99
    ops=$(awk -v L="$label" '
        $0 ~ "^=+=> perf-check: "tolower(L)" " { capture=1; next }
        capture && /ops\/sec/ { print $1 ? $1 : "0"; exit }' "$log" || true)
    # Prefer the structured summary "ops/sec" line printed by print_bench_summary.
    ops=$(awk -v L="$label" '
        BEGIN { in_summary = 0 }
        $0 ~ "^"L" summary" { in_summary = 1; next }
        in_summary && /ops\/sec/ { gsub(",", "", $2); print $2; exit }' "$log")
    p50=$(awk -v L="$label" '
        BEGIN { in_summary = 0 }
        $0 ~ "^"L" summary" { in_summary = 1; next }
        in_summary && /p50_ms/ { gsub(",", "", $2); print $2; exit }' "$log")
    p99=$(awk -v L="$label" '
        BEGIN { in_summary = 0 }
        $0 ~ "^"L" summary" { in_summary = 1; next }
        in_summary && /p99_ms/ { gsub(",", "", $2); print $2; exit }' "$log")
    echo "${ops:-0}|${p50:-0}|${p99:-0}"
}

snapshot_cpu() {
    local ps_pid ext_pid ps_cpu ext_cpu
    ps_pid=$(cat /tmp/autumn-rs/pids/ps.pid 2>/dev/null || cat /dev/shm/autumn-rs/pids/ps.pid 2>/dev/null || true)
    ext_pid=$(cat /tmp/autumn-rs/pids/node1.pid 2>/dev/null || cat /dev/shm/autumn-rs/pids/node1.pid 2>/dev/null || true)
    if [[ -n "$ps_pid" ]]; then ps_cpu=$(ps -o pcpu= -p "$ps_pid" | tr -d ' ' || true); fi
    if [[ -n "$ext_pid" ]]; then ext_cpu=$(ps -o pcpu= -p "$ext_pid" | tr -d ' ' || true); fi
    echo "${ps_cpu:-}|${ext_cpu:-}"
}

run_one() {
    local n="$1" cap="$2" rep="$3"
    echo "[R1] phase=$PHASE mode=$STORAGE_MODE N=$n cap=$cap rep=$rep"

    pushd "$REPO_DIR" > /dev/null
    ./cluster.sh clean
    AUTUMN_GROUP_COMMIT_CAP="$cap" ./cluster.sh reset "$REPLICAS" $CLUSTER_FLAGS
    popd > /dev/null

    local log="/tmp/perf_r1_${PHASE}_${STORAGE_MODE}_n${n}_cap${cap}_r${rep}.log"
    pushd "$REPO_DIR" > /dev/null
    local cpu_snapshot_fg=""
    (
        sleep 12  # mid-run snapshot, ~6s into the 10s write phase (after cluster start + write ramp)
        snapshot_cpu > "${log}.cpu"
    ) &
    local snap_bg=$!

    AUTUMN_GROUP_COMMIT_CAP="$cap" ./perf_check.sh $PERF_FLAGS --partitions "$n" 2>&1 | tee "$log"
    wait "$snap_bg" 2>/dev/null || true
    popd > /dev/null

    local result_write result_read
    result_write=$(extract_result "$log" "Write")
    result_read=$(extract_result "$log"  "Read")
    IFS='|' read -r w_ops w_p50 w_p99 <<< "$result_write"
    IFS='|' read -r r_ops r_p50 r_p99 <<< "$result_read"
    IFS='|' read -r ps_cpu ext_cpu <<< "$(cat "${log}.cpu" 2>/dev/null || echo '|')"

    echo "$PHASE,$STORAGE_MODE,$n,$cap,$rep,$w_ops,$r_ops,$w_p50,$w_p99,$r_p50,$r_p99,$ps_cpu,$ext_cpu," >> "$CSV"
    echo "[R1]  → write $w_ops ops/s (p99 $w_p99 ms), read $r_ops ops/s (p99 $r_p99 ms), ps_cpu=$ps_cpu% ext_cpu=$ext_cpu%"
}

main() {
    local cap_list="$CAPS"
    if [[ -z "$cap_list" ]]; then
        cap_list="default"   # means "do not set AUTUMN_GROUP_COMMIT_CAP; PS uses compiled default"
    fi
    for n in $PARTITIONS; do
        for cap in $cap_list; do
            for (( rep=1; rep<=REPS; rep++ )); do
                if [[ "$cap" == "default" ]]; then
                    run_one "$n" "3072" "$rep"     # record 3072 as the compiled default
                else
                    run_one "$n" "$cap" "$rep"
                fi
            done
        done
    done

    echo
    echo "[R1] Summary (median per (N, cap)):"
    python3 - <<PY
import csv, statistics
rows = list(csv.DictReader(open("$CSV")))
rows = [r for r in rows if r["phase"] == "$PHASE" and r["storage_mode"] == "$STORAGE_MODE"]
groups = {}
for r in rows:
    k = (int(r["partitions"]), int(r["group_commit_cap"]))
    groups.setdefault(k, []).append(r)
print("  N  cap   reps   w_med    r_med    w_p99_med")
for k in sorted(groups):
    rs = groups[k]
    w = statistics.median(float(r["write_ops"]) for r in rs)
    rd = statistics.median(float(r["read_ops"]) for r in rs)
    p99 = statistics.median(float(r["write_p99_ms"]) for r in rs)
    print(f"  {k[0]:<2} {k[1]:<5} {len(rs):<4}  {w:>8.0f}  {rd:>8.0f}  {p99:>8.2f}")
PY
}

main "$@"
```
