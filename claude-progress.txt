  1. New PS RPC `MSG_GET_DISCARDS` (0x48):
     - `GetDiscardsReq { part_id }` / `GetDiscardsResp { discards: Vec<(u64,i64)> }`
       in `crates/rpc/src/partition_rpc.rs`
     - `extract_part_id` extended with MSG_GET_DISCARDS arm
     - `handle_get_discards` in `crates/partition-server/src/rpc_handlers.rs`:
       snapshots sst_readers, calls `background::get_discards()`, filters
       via `background::valid_discard()` against live log_stream extent_ids,
       returns filtered (extent_id, bytes) pairs
     - Wired into `dispatch_partition_rpc` match

  2. `autumn-client info` enhancements
     (`crates/server/src/bin/autumn_client.rs`):
     - `Command::Info { json: bool, top: Option<usize>, part: Option<u64> }`
     - Parser: `--json`, `--top N`, `--part PID` (mutually exclusive for
       --top/--part)
     - Per-partition discard fetch: prefers `part_addrs[pid]` per F099-K,
       falls back to `ps_details[ps_id].address`
     - JSON output: `InfoSnapshot { nodes, extents, streams, partitions }`;
       each `InfoPartitionView` carries `discards: Vec<InfoDiscardEntry>`
     - Text output: log stream line gets `, discard: N ext / X pending`
       when non-empty; `--top` sorts partitions by live size; `--part`
       focuses the partitions section; `--top`/`--part` suppress
       nodes/extents/streams sections
     - `InfoNodeView`, `InfoDiskView`, `InfoExtentView`, `InfoStreamView`,
       `InfoPartitionView`, `InfoDiscardEntry`, `InfoSnapshot` — file-private
       Serialize types

  3. README.md updated with cluster info subsection + updated info table row
