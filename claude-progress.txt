TaskStatus: completed
Task scope: F118 — open extents on EC streams misrouted as EC. PS panicked
post-split with `range start index 93582868 out of range for slice of length
8388569` on `client.rs::read_with_layout → ec_subrange_read`, and `autumn-
client info` labelled open extents as `EC(3+1)` even though they were still
4-way replicated.

Current summary:
  Symptom: live cluster after split. PS `part-20` (the new partition) panics on
  read; info command shows open extents 21/23/24 as `EC(3+1), data=[1,3,5],
  parity=[7]` despite `(open)` tag.

  Root cause: the manager's `stream_alloc_extent` (rpc_handlers.rs:792-803)
  pre-fills `parity` with the M target nodes the moment an EC-stream extent
  is allocated. The data plane writes the full payload to every K+M replica
  until the `ec_conversion_dispatch_loop` runs `apply_ec_conversion_done` on
  a *sealed* extent. Anything that branches on `parity.is_empty()` to
  distinguish replication-vs-EC misroutes pre-conversion / open extents:
    1. `client.rs::read_with_layout` → `ec_subrange_read`: panics on
       `shard_size = shard_size(sealed_length=0, K) = 0`.
    2. `extent_node.rs::run_recovery_task` → `run_ec_recovery_payload`:
       would corrupt EC recovery (data is replicated, not EC-shaped).
    3. `autumn_client.rs::Info`: misleading display.

  The manager's own EC conversion dispatch already used the correct marker
  (`ex.original_replicates != 0` at recovery.rs:377), but `original_replicates:
  u32` was confusingly named — only the zero / non-zero distinction was ever
  consumed. Renamed to `ec_converted: bool` for clarity.

Fix:
  1. Renamed `original_replicates: u32` → `ec_converted: bool` in
     `MgrExtentInfo` (rpc/src/manager_rpc.rs) + `ExtentInfo`
     (stream/src/extent_rpc.rs). Wire-format / etcd-schema change.
  2. Init sites in `manager/src/rpc_handlers.rs` (2 locations): set
     `ec_converted: false`.
  3. `apply_ec_conversion_done` (manager/src/recovery.rs): drop the
     `original_replicates: u32` parameter (and the caller's
     `orig_replica_count`); set `ex.ec_converted = true`.
  4. EC conversion dispatch filter (recovery.rs:377): `ex.ec_converted`.
  5. Read-path dispatch (`client.rs::read_with_layout`): `if
     ex.ec_converted` — the panic site.
  6. Recovery dispatch (`extent_node.rs::run_recovery_task`): `if
     !extent_info.ec_converted` for the replication path.
  7. Info display (`autumn_client.rs::Info`): render `EC(K+M)` only when
     `ec_converted`; otherwise render `replicas=[…replicates ++
     parity…]` so all K+M nodes that physically hold the replicated
     payload show up.
  8. mgr↔local conversion helpers: copy through.
  9. Tests (`ec_failover.rs`, `update_stream_ec.rs`): assertion switched
     from `!ext.parity.is_empty()` to `ext.ec_converted`.
  10. `crates/stream/CLAUDE.md` Programming Note 16: codify the rule.
