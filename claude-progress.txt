Task scope: Drop `decide()`/`AUTUMN_TRANSPORT` env probe; require explicit
  `--transport tcp|ucx` CLI flag on every binary. Add `ucx` feature
  forwarding on the autumn-server crate so binaries actually link UCX
  when built with `--features autumn-server/ucx`.

Current summary:
  Twelve files changed in this session.

  crates/transport/src/probe.rs
    - Replaced. `decide()`, `Decision`, `probe_ucx()`, `RDMA_TRANSPORTS`,
      and the `open_memstream` UCX context probe are all gone. The file
      now exposes only `parse_transport_flag(&str) -> Result<TransportKind, &str>`,
      a small CLI-string parser accepting "tcp" or "ucx".

  crates/transport/src/lib.rs
    - `pub use probe::{decide, Decision}` → `pub use probe::parse_transport_flag`.
    - `init()` → `init_with(kind: TransportKind)`. The choice now comes from
      the caller. Panics if `TransportKind::Ucx` is requested but the binary
      was built without the `ucx` feature.
    - `current_or_init()` lazy fallback defaults to TCP (was: read env via
      `init()` → `decide()`).
    - `check_listen_addr` doc comment updated from "after init()" to
      "after init_with()".

  crates/server/Cargo.toml
    - Added `ucx = ["autumn-transport/ucx"]` feature so
      `cargo build --features autumn-server/ucx` actually links UCX into
      the four server binaries. Pre-fix, `--features autumn-transport/ucx`
      at the workspace level didn't propagate into autumn-server, so
      autumn-client and autumn-manager-server were silently TCP-only.

  crates/server/src/bin/manager.rs
  crates/server/src/bin/extent_node.rs
  crates/server/src/bin/partition_server.rs
  crates/server/src/bin/autumn_client.rs
    - All four binaries: added `--transport tcp|ucx` CLI arg (default `tcp`),
      moved `parse_args()` before transport init, replaced
      `autumn_transport::init()` with `init_with(args.transport)`.
    - partition_server.rs: also added `--transport <MODE>` to `--help` text.
    - autumn_client.rs: `--transport` is a global arg (alongside `--manager`).

  crates/partition-server/benches/ps_bench.rs
    - Removed the `std::env::var("AUTUMN_TRANSPORT")` default and the
      `std::env::set_var("AUTUMN_TRANSPORT", &c.transport)` write-back.
      Now defaults to "tcp" and calls `autumn_transport::init_with(kind)`
      explicitly at the end of `Config::from_args()`.

  crates/transport/tests/probe.rs
    - Replaced the `decide()` env-var test matrix with two simple tests
      for `parse_transport_flag` accepting "tcp"/"ucx" and rejecting
      everything else.

  crates/transport/tests/init.rs
    - Updated the single `init() → init_with(TransportKind::Tcp)` test to
      reflect the new explicit-init API.

  cluster.sh
    - Added a TRANSPORT validation block at top: reads AUTUMN_TRANSPORT
      (default "tcp"), accepts only tcp|ucx, then `unset AUTUMN_TRANSPORT`
      so it doesn't leak to child binaries.
    - `wait_port` switched from `${AUTUMN_TRANSPORT:-auto}` to `$TRANSPORT`.
    - Added `--transport "$TRANSPORT"` to every binary launch:
      manager, extent-node (4 disk × shard variants), PS, and the three
      autumn-client invocations (register-node, format, bootstrap).

  perf_check.sh
    - Build command flag: `--features autumn-transport/ucx` →
      `--features autumn-server/ucx` (Item F: feature now lives on the
      autumn-server crate which actually owns the binary build).
    - `AUTUMN_TRANSPORT="$mode" "$AC" perf-check ...` →
      `"$AC" --transport "$mode" perf-check ...`. The cluster.sh invocation
      keeps its `AUTUMN_TRANSPORT="$mode" bash cluster.sh start 3` form;
      cluster.sh consumes the env var internally and translates it to
      `--transport` flags.
