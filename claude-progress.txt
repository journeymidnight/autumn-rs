Task scope: F119 — gallery example: transcode uploaded video to single-bitrate
HLS (M3U8 + .ts) asynchronously after upload, store segments in Autumn KV,
play with hls.js (Safari uses native HLS), drop the original after transcode,
and surface transcode state in the grid via /transcode-status polling.

Current summary:
  Goal: replace direct MP4 byte streaming with HLS so video playback uses a
  standard chunked protocol (browser/CDN-friendly, hls.js-compatible).

  Approach (approved A in brainstorming):
    - Async background transcoding via compio::runtime::spawn; FFmpeg
      runs in spawn_blocking, pulls source from /get/ via HTTP Range.
    - Single-bitrate libx264 / aac, hls_time=4, no ABR.
    - Store HLS segments in KV under .hls/<name>/{index.m3u8,seg*.ts};
      thumbnail at .thumb/320/<name> (same prefix as image thumbs);
      delete original after transcode completes.
    - In-memory transcode status map (Rc<RefCell<HashMap<...>>> — single
      compio thread, no Arc/DashMap); KV fallback (.hls playlist exists ⇒
      done; original exists w/o playlist ⇒ queued) handles server restart.
    - 409 Conflict on same-name upload while transcoding.
    - Front-end uses hls.js@1 from jsDelivr; cells render a "转码中…"
      placeholder + spinner with backoff polling (2 s ×5 → 5 s ×5 → 10 s).

Verification:
  - cargo build --release -p gallery -p autumn-server -p autumn-client: clean.
  - End-to-end on 1-node cluster (./cluster.sh start 1):
      * upload sample.mp4 (8 s testsrc) → /transcode-status flips to done in
        ~500 ms → /hls/sample.mp4/index.m3u8 returns valid #EXTM3U
        (application/vnd.apple.mpegurl) → /hls/sample.mp4/seg000.ts returns
        181 KB of video/mp2t → /thumb/sample.mp4 returns image/jpeg →
        /get/sample.mp4 → 404 → /list/ shows sample.mp4.
      * sample.mov upload → same flow.
      * red.png upload → unchanged behavior.
      * DELETE /del/sample.mp4 → list no longer shows it; hls + thumb
        keys all 404.
      * concurrent same-name upload while transcoding → HTTP 409
        ("transcoding in progress for big.mp4").
      * restart gallery → status endpoint returns done via KV fallback.
      * kill gallery mid-transcode → on restart, recover_pending_transcodes
        re-enqueues the orphan and the transcode completes.
  - In-browser verification (hls.js attach, autoplay, lightbox cleanup) is
    covered by the README's manual steps; not exercised in this session.
