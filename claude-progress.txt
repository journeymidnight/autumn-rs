Current summary:
  One file changed.

  crates/transport/src/ucx/endpoint.rs
    - Removed `impl Drop for UcxEp` entirely (was: zero out raw + call
      `ucp_ep_close_nbx` with `UCP_EP_CLOSE_FLAG_FORCE`). The FORCE call
      was a noisy no-op: UCX rejects FORCE on EPs created with the
      default `UCP_ERR_HANDLING_MODE_NONE`, returning
      `UCS_ERR_INVALID_PARAM (-5)` synchronously and *not* closing the EP.
      So the call achieved nothing functional — just produced one
      `tracing::warn!` per EP teardown.
    - Updated the module-level "EP lifetime" doc to spell out why we
      now skip the close call entirely, and which alternatives were
      ruled out:
        * FORCE close requires PEER mode (rejected with -5 otherwise).
        * Graceful FLUSH-mode close deadlocks the loopback test (both
          peers in the same compio runtime block each other's ack).
        * `UCP_ERR_HANDLING_MODE_PEER` (which would let FORCE succeed)
          enables UCX's RC-layer dead-peer detection — under high
          fan-out write load (16 client threads × 16 inflight against
          8 PS workers) rc_mlx5 RNR/timeout fires *mid-test* and tears
          down live EPs (verified empirically: write ops/s collapsed
          from 16 k → 0 around the 8-second mark, read ops/s stayed
          at 0). Worse than a teardown warning.
      Skipping the call leaks the C-side EP in the worker context until
      `ucp_worker_destroy` (thread/process exit). For autumn-rs this is
      fine: autumn-rpc::pool and autumn-stream::conn_pool reuse one EP
      per (peer-address, thread) pair for the life of the process, so
      EPs do not accumulate at runtime.
    - listener.rs unchanged (server-side EPs share the same Rc<UcxEp>
      Drop path — fix at the one site covers both directions).
